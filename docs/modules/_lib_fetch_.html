Const my_fetch

my_fetch: (arg: string) => Promise<any> = (process !== undefined) ? node_fetch.default : fetch
  • Defined in lib/fetch.ts:79

The effective fetch implementation run by the rest of the code.

There is no default fetch implementation for node.js, hence the necessity to import 'node-fetch'. However, if the code runs in a browser, there is an error message whereby only the fetch implementation in the Window is acceptable.

This variable is a simple, polyfill like switch between the two, relying on the existence (or not) of the process variable (built-in for node.js).

Const check_Web_url

  • check_Web_url(address: URL): URL
    • Defined in lib/fetch.ts:34

    Basic sanity check on a URL supposed to be used to retrieve a Web Resource.

    The function returns a (possibly slightly modified) version of the URL if everything is fine, or a null value if the input argument is not a URL (but should be used as a filename).

    There might be errors, however, in the case it is a URL.

    The checks are as follows:

    1. Check whether the protocol is http(s). Other protocols are not accepted (actually rejected by fetch, too);
    2. Run the URL through a valid-url check, which looks at the validity of the URL in terms of characters used, for example;
    3. Check that the port (if specified) is in the allowed range, i.e., > 1024;
    throws

    if address pretends to be a URL, but it is not acceptable for some reasons.

    Parameters

    • address: URL

    Returns URL

    • the URL itself (which might be slightly improved by the valid-url method) or null if this is, in fact, not a URL
