Examples for JSON Collection Configuration files

These are just some examples for (JSON) Collection Configuration files. They are here for illustrative, and possibly testing purposes.

  • Collection examples:
    • audiobooks.json: Publication Manifest and Audiobooks;
    • json-ld.json: JSON-LD 1.1;
    • vc.json: Verifiable Credentials Specification and Use Cases;
    • epub.json: Epub 3.3 Editors’ Draft;
    • For testing:
    • config_err.json: an invalid JSON file, used for testing the error reports;
    • json-ld_test.json: the JSON-LD 1.1 specification family, including the Streaming JSON-LD Note; sources taken from the editors' drafts, pre-processed through ReSpec with the status set to REC and an explicit data set (this is for testing purposes, the 'real' JSON-LD 1.1 collection should rely on the latest published versions);
    • single.json: a collection with a single entry, namely the JSON-LD 1.1 syntax document specification (this is for testing purposes for a single document as a collection; the 'real' version should simply use the URL of the latest published version directly);
